# Agent Contract Testing

Two-tier test system that validates agent prompt files against the STATUS contract defined in `docs/agent-architecture.md`. The static linter is a zero-cost always-on CI gate; the live test is opt-in and validates actual proxy responses against the same contract.

- **From Session b1731578:** Static linter (`test/agent-contract-static.test.js`, 94 assertions): parses every `agents/*.md` file and validates (1) model: frontmatter matches filename, (2) STATUS: COMPLETE declared for worker agents, (3) RECUSE block present with correct RECOMMEND escalation target, (4) extra fields (LINT_ITERATIONS, ITERATIONS) declared, (5) fail-closed: any agent in `agent_to_capability` without a roster entry fails immediately. Design principle: new agents cannot silently drift from spec — adding an entry to the config without a roster entry is a hard failure.
- **From Session b1731578:** Spec-gap tier system in the roster: `needsRecuse: 'fail'` for agents with declared RECUSE blocks (implementer, reviewer-fix, test-writer, scaffolder, converger, security-reviewer), `needsRecuse: 'warn'` for agents with known spec gaps (integrator, doc-writer, discovery-advisor, planner-local), `needsRecuse: 'exempt'` for read-only agents (explorer), `warnOnly: true` for judge/orchestrator/utility tiers (only model: field checked). This prevents false test failures from known-incomplete agents while tracking them explicitly.
- **From Session b1731578:** Special roster entries: `security-reviewer` has `recuseException: true` (RECUSE present but RECOMMEND must be absent — no cascade target, judge-strict hard_fail), `uplift-decider` uses `special: 'uplift-decider'` (VERDICT contract with CLOUD_CONFIDENCE + RATIONALE, must NOT contain STATUS: RECUSE), `implementer-cloud` and `test-writer-cloud` RECOMMEND to `judge` (sentinel — no agent to dispatch to, orchestrator marks as escalated:judge_blocked).
- **From Session b1731578:** Live contract test (`test/agent-contract-live.test.js`): opt-in via `C_THRU_LIVE_AGENT_TESTS=1`, POSTs each agent's system prompt to a running proxy at `CLAUDE_PROXY_URL` or `CLAUDE_PROXY_PORT`, validates that responses contain parseable STATUS blocks with valid STATUS values (COMPLETE|PARTIAL|ERROR|RECUSE), valid CONFIDENCE (high|medium|low), SUMMARY, and agent-specific fields. Strips `<think>` blocks before parsing. 12 agents in LIVE_ROSTER with tailored userMessages and per-agent `extraChecks`.
- **From Session b1731578:** Roster-CONFIG synchronization: the static test cross-references its ROSTER object against `CONFIG.agent_to_capability` — any agent name present in config but missing from the roster causes a hard failure. This ensures the test stays in sync with agent configuration, not just with agent files on disk.

- **From Session b1731578:** Behavioral gap analysis post-implementation: the live test only validates structural conformance, not behavioral correctness. Identified 6 uncovered paths: (1) no PARTIAL/crisis trigger — no test gives implementer a task with a deliberate blocker; (2) no RECUSE trigger — no test gives scaffolder a task requiring design decisions; (3) no ITERATIONS/LINT_ITERATIONS verification; (4) no CONFIDENCE rubric test (task with unverifiable criteria should produce low/medium); (5) no agent-specific output validation (discovery-advisor should return `GAPS: N`; planner-local should return `VERDICT: ready|done`). Three testing dimensions identified: (1) **prompt correctness** — fixture digests per agent, assert STATUS value matches; (2) **routing correctness** — already covered by proxy unit tests; (3) **behavioral quality** — requires a judge, not a unit test. Recommendation: fixture-based prompt correctness is the highest-value gap to fill next, by creating one realistic digest per worker agent and asserting the expected STATUS value.

- **From Session b1731578:** Fixture design constraint for planned `test/agent-contract-behavioral.test.js`: every behavioral test fixture must append `shared/_worker-contract.md` to the digest written to tmpdir before dispatch. Without it, agents lack their recusal rubric and response structure rules and will not behave correctly — e.g. will not RECUSE on ambiguous tasks. This makes `shared/_worker-contract.md` a required dependency for any behavioral test, not just for production dispatch. (`shared/` was chosen over `agents/` to avoid inflating the fail-closed roster count in the static linter.)

- **From Session a183dfe6:** Post-ship corrections to both test files: (1) static test: `reviewer-fix` entry in ROSTER renamed to `wave-reviewer` (D4 — agent file was renamed in PR #40); `implementer-cloud` and `test-writer-cloud` also added to `needsRecuse: 'fail'` tier — the b1731578 bullet above lists only 6 agents but current ROSTER has 8 fail-tier entries (adding the two cloud variants); (2) live test: `MAX_TOKENS` bumped from 800 → 1200 (some judge-tier agents were truncating STATUS blocks under the lower limit); (3) live test: 401/403 responses now cause an agent-level `skip` rather than `fail` — cloud backends may not be configured in all environments. These are maintenance fixes to track the wave-reviewer rename and tune test robustness.

- **From Session b1731578:** Behavioral test completed: `test/agent-contract-behavioral.test.js` (953 lines at commit 8520d35; grew to 998 lines after session 75859eff hardening). Guard: `C_THRU_BEHAVIORAL_TESTS=1`. Subset filter: `BEHAVIORAL_ONLY=<agent>`. Covers 14 agents across 3 phases: Phase 1 high-confidence (discovery-advisor, explorer, uplift-decider), Phase 2 worker agents, Phase 3 cloud-tier agents. Final result: 36 tests, 36 passed, 0 failed, 4 skipped. Key design decisions: (1) **BEHAVIORAL_WORKER_CONTRACT** strips the post-work linting section — without stripping, agents RECUSE immediately because they can't run `node --check` in a raw API call; (2) **/no_think suffix** appended globally to worker digest messages — Qwen3 extended thinking exhausts the token budget before producing text output, causing empty responses; (3) **`noThink: false`** on inline key-value agents (discovery-advisor, planner-local) — `/no_think` disrupts their format parsing when appended; (4) **`assertWorkSection(..., advisory=true)`** for doc-writer and scaffolder — local models return valid STATUS but inconsistently follow the response structure section format; (5) **per-entry `timeoutMs: 300000`** for high-token worker agents on thinking models. 4 skips: security-reviewer/implementer-cloud/test-writer-cloud (HTTP 401, cloud backend not configured), planner-local (returns empty text even with no_think off — routing inconsistency with local model). 1 advisory: uplift-decider returns `VERDICT: restart` instead of `accept` on partial-satisfying input — local judge-tier model is systematically conservative. Proxy lifecycle gap: no setup/teardown in test suite; proxy must be started manually via `CLAUDE_PROXY_PORT=N node ~/.claude/tools/claude-proxy` before running and killed after.

- **From Session 75859eff:** Test suite hardening plan (9 fixes) identified from running real LLM calls through behavioral suite. Critical fragility: `buildWorkerDigest` returned a filesystem **path** as the user message — raw `/v1/messages` API calls have no tool use, so models can't read the file and RECUSE systematically. Fix: return file **contents** as user message; keep writing to disk for reference only. Same issue applies to `planner-local` which sent current.md and wave_summary as file paths instead of inline contents. Rule: never send a file path as a user message in a raw-API behavioral test.
- **From Session 75859eff:** Routing verification gap: behavioral tests passed even when the proxy wasn't doing agent-name resolution, because `postMessages` dropped response headers and nothing asserted `x-c-thru-resolved-via`. Fix: update `httpJson`/`postMessages` to return `headers`, then assert per-request: (1) header must be present (proves c-thru path); (2) `served_by !== agentName` (proves alias-graph traversal); (3) `via.capability === entry.expectedCapability` (optional strict validation). Add `expectedCapability` to each ROSTER entry for precise routing assertion.
- **From Session 75859eff:** Parser divergence: `agent-contract-live.test.js` stripped `<think>` only; `agent-contract-behavioral.test.js` also normalized pipe-separated STATUS blocks. A Qwen3 response passing behavioral could fail live. Fix: extract unified `parseStatusBlock(text)` to `test/helpers.js` — strips `<think>…</think>`, replaces `|KEY:` with newline+`KEY:`, then parses all `KEY: value` pairs. Both test files import from helpers; local definitions deleted.
- **From Session 75859eff:** Skip classification: cloud 401/403 (no `ANTHROPIC_API_KEY`) vs structural failures (no STATUS block, timeout, socket error, missing file) were both labeled as `skip`, making CI green on real regressions. Fix: split into `skippedExpected` (HTTP 401/403 on cloud tiers) and `skippedUnexpected` (everything else). CI gate: `process.exit(failed || skippedUnexpected ? 1 : 0)`. Additional hardening: `stripBehavioralContract` must throw (not silently no-op) if the `## Post-work linting` section boundary moves; centralize per-capability timeouts in `TIER_TIMEOUTS_MS` map in helpers.js to eliminate per-entry magic numbers; SIGINT tmpDir cleanup via `registerTmpDir` + `installExitHandlers` pattern.

- **From Session a553415d:** Static test ROSTER extended for new planner signals shipped in PR #42: `discovery-advisor` and `explorer` assert `TEST_FRAMEWORKS` as an `extraFields` entry; `plan-orchestrator` asserts `TEST_FRAMEWORKS`, `COMPLEXITY`, and `MIGRATION_REQUIRED`. Implementation fix: `extraFields` check was hoisted above the `warnOnly` short-circuit — without this, `warnOnly: true` agents would silently skip all `extraFields` assertions, making the new field coverage effectively inert for those entries.
