# Test Suite Hardening

Seven concrete fragilities identified after running the behavioral test suite (`test/agent-contract-behavioral.test.js`) against real LLM calls. These are post-implementation findings that form a P0-P3 hardening plan for the test suite.

- **From Session b1731578:** 7 fragilities (P0-P3 plan): **(P0) Worker agents always RECUSE** — `buildWorkerDigest` writes fixture to a file path and sends the path as the user message; worker agents have no tool use in raw API calls and cannot read the file, causing systematic RECUSE. Fix: inline the digest content directly rather than referencing a path (same fix that resolved the explorer template-echo issue). **(P0) No routing proof** — tests pass even if the proxy isn't doing `agent_to_capability` resolution; the proxy emits `x-c-thru-resolved-via` on every capability response with `served_by`, `capability`, `tier`, and `mode`, but `postMessages` drops response headers and nothing asserts the header exists. Fix: capture response headers in `postMessages` and assert the header is present with the expected `capability` and `served_by` fields. **(P0) Two divergent parsers** — `agent-contract-live.test.js` strips `<think>` only; `agent-contract-behavioral.test.js` also normalizes pipe-separated STATUS blocks. A Qwen3 response that passes behavioral may fail live. Fix: extract a shared `parseStatusBlock()` to `test/helpers.js`. **(P0) Contract strip has no assertion** — the regex stripping the linting section from `shared/_worker-contract.md` silently no-ops if the file is reorganized, causing downstream RECUSE with no diagnostic. Fix: assert the stripped content differs from the original. **(P1) `/no_think` in wrong scope** — appending `/no_think` to user messages corrupts KV-format agents (discovery-advisor, planner-local) whose parsers treat the suffix as part of the payload; the correct placement is the system prompt for non-KV agents. **(P2) tmpDirs leak on SIGINT** — no signal handler; Ctrl-C mid-run leaves `c-thru-beh-*` dirs under `os.tmpdir()`. Fix: `process.on('SIGINT', cleanup)`. **(P2) Skip semantics are ambiguous** — cloud 401, no-STATUS-block, and missing agent files all produce identical `skip` lines; a proxy returning 200 with no body still shows a "green" run. Fix: classify skips as `expected-skip` vs `unexpected-skip` and gate CI on unexpected-skip count.

- **From Session b1731578:** `x-c-thru-resolved-via` as the routing verification mechanism: the proxy sets this response header on every capability request (agent-name dispatch). Its value is a JSON object: `{"capability": "workhorse", "profile": "workhorse", "served_by": "claude-sonnet-4-6", "tier": "64gb", "mode": "connected", "local_terminal_appended": false}`. Currently `postMessages` in the behavioral test runner discards all response headers. To prove routing correctness, the test must: (1) capture the header in the HTTP response callback, (2) JSON-parse it, (3) assert `capability` matches the expected alias from `agent_to_capability`, and (4) assert `served_by` is a non-empty string. This is a stronger routing proof than asserting STATUS structure alone — it verifies the proxy understood the agent name and applied the correct resolution chain.

- **From Session 75859eff:** All P0 fragilities resolved and shipped. `buildWorkerDigest` now returns inline file contents (not a path) — worker RECUSE eliminated. `planner-local` similarly inlines `current.md` and `wave-summary.json`. `postMessages` returns response headers; `runTest` asserts `x-c-thru-resolved-via` is present, `served_by !== agentName`, and `capability === entry.expectedCapability` — routing now has a hard proof, not just structural conformance. Shared `parseStatusBlock()` extracted to `test/helpers.js` (strips `<think>`, normalizes pipe-separated KEY: blocks) — both test files import it; local copies deleted. Skip semantics split into `skippedExpected` (HTTP 401/403 on cloud tiers) vs `skippedUnexpected` (no STATUS block, timeout, socket error, structural failure); CI exits 1 on any unexpected skip. `/no_think` moved from user-message suffix to system prompt via `QWEN3_TIERS` set — eliminates KV-format corruption and removes need for per-entry `noThink` flags. `stripBehavioralContract` now throws if the `## Post-work linting` boundary moves (was silently no-oping). SIGINT tmpDir cleanup via `registerTmpDir` + `installExitHandlers`. 14 ROSTER entries all have `expectedCapability` field.
