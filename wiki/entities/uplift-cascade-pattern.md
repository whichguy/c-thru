# Uplift-Cascade Pattern

A three-tier decision cascade where a local worker produces output, verify.json gates on tests+syntax, and an `uplift-decider` (cloud judge, ~200 tokens) chooses accept/uplift/restart. The key design insight: **when verdict is `restart`, the cloud implementer never sees the local output** ‚ÄĒ anchoring is prevented by context omission, not by prompt hint. Generalizes across four agent roles (implementer, reviewer-fix, test-writer, planner) using a uniform decider schema with `context_kind` branching.

- **From Session 1558f542:** Research-grounded design: speculative cascades (Google, 2-3√ó speedup), GATEKEEPER/ReDAct (confidence-based deferral), CRITIC (tool-grounded critique beats self-reflection), "More Agents" (2-3 independent samples + vote, diminishing returns past 3), TACL survey (same-model self-correction is fragile ‚ÄĒ external critics/oracles are reliable). Deliberate no-gos: multi-agent debate (vote is simpler), self-reflection without tool grounding, N‚Č•4 parallel samples, separate PM/Architect persona, replacing review-plan with one-shot `/speckit.analyze`.
- **From Session 1558f542:** Confidence-graded STATUS (E3): workers self-report `CONFIDENCE: high|medium|low` + `UNCERTAINTY_REASONS` with a backward-looking, evidence-based rubric. High = reused existing patterns + success_criteria map directly + can state satisfaction in one sentence each + made no assumptions. Medium = improvised pattern / interpreted criteria / guessed at API / added unneeded error handling. Low = unfamiliar domain / missing resources / ambiguous description / couldn't find calling site. Miscalibration tracked: over-confident (high but verify fails), under-confident (low but verify passes), uncertain (medium, passes half ‚ÄĒ actually correct state).
- **From Session 1558f542:** Anchoring prevention: `restart` verdict means the cloud implementer's prompt deliberately omits `local_output_path` ‚ÄĒ it solves from the digest alone. Grep-able invariant: `mode=clean prompts NEVER contain 'draft', 'prior', 'attempt', or 'previous'`. Enforced by unit test on orchestrator prompt builder, not just convention. Two decider biases guarded: always-restart (wastes local work, drives up cloud cost) and always-uplift (cloud defers to existing draft, anchoring persists). Decider prompt forces explicit criteria per verdict + one-line rationale.
- **From Session 1558f542:** Generalization across four agent roles via uniform decider schema with `context_kind: implementation|review|tests|plan`. Implementer cascade: local‚Üíverify‚Üídecider‚Üí{accept|uplift-cloud|restart-cloud}. Reviewer-fix cascade: after round 2 with no convergence, decider chooses {accept (cosmetic findings) | uplift-cloud (cloud reviewer continues with context) | restart-cloud (fresh from digest+source)}. Test-writer cascade: local test-writer‚Üírun tests‚Üídecider‚Üí{accept|uplift (add missing cases)|restart (redesign strategy)}. Planner cascade: already exists as dep_update/outcome_risk transition ‚ÄĒ the existing wave-transition logic is a prior instance of this pattern at a different granularity.
- **From Session 1558f542:** Converger as first-class logical role: new agent `converger` (local tier) with three invocation sites ‚ÄĒ (1) Phase 1 discovery reconciliation after explorer fan-out (flag disagreements, produce reconciled context), (2) multi-attempt selection for E4 best-of-N (compare N implementations, select winner by test-pass + majority similarity), (3) wave-summary aggregation (structured replacement for orchestrator's prose scan). Named and explicit makes convergence tunable (swap tier, cap timeouts) and the contract visible.
- **From Session 1558f542:** Implementation sequencing: E3 (confidence) ‚Üí E1 (speculative cascade, now uplift-cascade) ‚Üí E2 (converger) ‚Üí E6 (test-first path) ‚Üí E5 (local pre-reviewer, cheap 7B screen before reviewer-fix) ‚Üí E4 (best-of-N with converger) ‚Üí E7/E8 (situational). Each ‚Č§1 wave. Measurement: `cloud_hops_per_plan`, `wall_clock_per_wave`, `reviewer_fix_rounds_per_item`, `human_escalation_count`. Keep only changes that move ‚Č•1 metric 20%+ without degrading another. Per-wave cascade budget: `WAVE_CLOUD_ESCALATION_BUDGET=3` (default), prevents runaway cost.
- **From Session 1558f542:** Observability: `$wave_dir/cascade/<item-id>.jsonl` with structured events per decision step (local_worker, decider, cloud_worker) including verdict, rationale, duration. Per-wave rollup in `wave-summary.md` with stats (local_only_accepts, decider_invocations, verdict distribution, cloud_hops, budget_used). Enables tuning without reading individual traces.

- **From Session 1558f542:** Self-review (¬ß19) identified 12 weaknesses in the cascade proposal. Most critical: W2 (CONFIDENCE is load-bearing but uncalibrated ‚ÄĒ if the signal is unreliable, cascade decisions are noise), W3 (decider adds net-new cloud hop; math unclear on whether cost offsets savings), W6 (premature generalization to 4 agent roles before implementer cascade is proven ‚ÄĒ YAGNI), W7 (anchor-prevention grep assertion fragile ‚ÄĒ use whitelist construction not denylist for clean-mode prompts). Result: scope contracted to Wave 1 only (CONFIDENCE field + logging hook) with explicit kill-switch gate. See [[cascade-scope-contraction]].
- **From Session 1558f542:** Wave-2 target architecture (build only if Wave-1 gate passes): every local worker output is a candidate draft; when verify fails or CONFIDENCE is low, `uplift-decider` (cloud judge, ~200 tokens) decides accept/uplift/restart. New agents: `uplift-decider.md`, `implementer-cloud.md` (two prompt modes), `converger.md` (aggregates parallel outputs). `escalation_policy` per item: `default | always_cloud | never_cloud`. `WAVE_CLOUD_ESCALATION_BUDGET=3` per wave. Migration: CONFIDENCE field optional for 1 release cycle (absent ‚Üí treated as `medium`). Deferred expansion proposals catalogued as E2-E8 in plan Appendix B.
